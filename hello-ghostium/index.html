Hello, Ghostium!

Alex Martin

I stumbled on a great new theme for my blog: Ghostium! A big thank you to Oswaldo Acauan for developing and maintaining it.

I had one small bug in converting Ghostium with my static generator. In the sidebar, on my blog pages, the "Home" link routed back to the page you're currently on instead of back to the blog's homepage. I was able to track the issue back to navigation.hbs, one of the config files located in ghost/content/themes/ghostium/partials/custom.

By default, Ghostium sets your home link on each page like so: 

<a href="/" data-pjax>  
    <i class="fa fa-home"></i>Home
</a>

This works fine for a live site, but when your static generator runs it interprets "/" as "current directory" instead of "site root directory", leading to the home link being rerouted to the current page.

The fix is super-simple: replace "/" with "{{@blog.url}}". Your new code will look like this: 

<a href="{{@blog.url}}" data-pjax>  
    <i class="fa fa-home"></i>Home
</a>

This is around line 29 in my navigation.hbs. The variable @blog.url relies on the site url set in your config.js, so make sure you've set that up before trying this!

With this fix in place I can deploy direct from a live instance to my gh-pages branch and see almost instant service to my custom domain.
