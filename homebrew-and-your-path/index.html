Homebrew & your PATH

Alex Martin

This is an addendum to my previous post addressing the most common error I see with new Homebrew installations.

Try something with me: type echo $PATH into your terminal and press enter. See all that mess that comes out? That's your PATH: a list of directories that your terminal expects to contain any command you might ever use. If you're a more file-oriented person, you can find your operating system's default PATH implementation by opening /etc/paths with your favorite text editor (careful - don't edit it!).

When you submit a command, say git --version, your computer searches each of those directories in the order they're listed until it finds a program called git. It then asks git what to do with --version. git will take whatever it's handed by the OS, process it, and return its output to your terminal for you to do with as you please.

This can cause an issue when you have multiple versions of software on your system. Imagine you have two versions of Git and your PATH looks like this: /old/git:/new/git. No matter how often you update that /new/git, the /old/git will always be used since it shows up first.

The common way to fix this is by appending newer directories to the front of your PATH, like this: 

    export PATH="/new/dir/route:$PATH"

This sets your PATH as equal to the new directory followed by your original PATH list.

The first time you run brew doctor, you'll probably see an error regarding /usr/bin & your PATH. If you see this error, run the following command in your terminal: 

    'export PATH="/usr/local/bin:$PATH"' >> ~/.bash_profile && source ~/.bash_profile

This will ensure your terminal checks Homebrew's installed & updated apps before it checks an older & possibly outdated folder.

This directive is written to the .bash_profile file in your home directory. If you need to remove it later, open ~/.bash_profile and delete the command. Do be careful - your profile will probably contain lots of other nuts & bolts and if you're using this tutorial you may not yet know what it all means.

If you're a purist regarding your PATH, or if you need support for the older XCode versions of certain programs, take a look at this answer on SuperUser. This is a much more complicated solution than I need, but it may help you if defaulting to Homebrew's versions causes issues on your system.

Let me know if this doesn't help clear up your PATH warnings! @ADotMartin
